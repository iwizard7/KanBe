# KanBe - Kanban Board Application

Профессиональное и эстетичное веб-приложение для управления задачами по методологии Kanban.

## Новые возможности (v2.5) — UI Redesign 🎨

### ✨ Визуальное обновление "Deep Nocturnal"
- **Премиальный дизайн**: Полностью переработан интерфейс с использованием эстетики "Deep Nocturnal" (темная тема) и "Snow Prism" (светлая тема).
- **Новая типографика**: Переход на современный шрифт **Outfit** для лучшей читаемости и премиального вида.
- **Улучшенный Glassmorphism**: Глубокое размытие (40px) и "умные" тени создают эффект многослойности.
- **Интерактивность**: Добавлены мягкие свечения (glow) при наведении на задачи и плавные переходы между темами.
- **Truly Light Mode**: Светлая тема полностью очищена от темных элементов для максимальной легкости.

### 🗂️ Управление проектами (v2.4)
- **Категоризация по проектам**: Группируйте задачи по проектам для лучшей организации.
- **Цветовое кодирование**: Присваивайте проектам уникальные цвета, которые отображаются на карточках в виде бейджей.
- **Глобальный фильтр**: Мгновенное переключение между проектами через выпадающий список в шапке.
- **Swimlanes по проектам**: Группировка доски по горизонтальным дорожкам-проектам (введите `#project`).
- **Управление в настройках**: Создание, редактирование и удаление проектов с автоматическим подсчетом задач.

### 🛠️ Инсталлятор и Администрирование (v2.3)
- **Интерактивный установщик**: Скрипт установки теперь автоматически проверяет занятость портов и предлагает выбрать другой, если 3000 занят.
- **Расширенное логирование**: Полная информация об IP-адресах при входе (успешном/неудачном) и попытках несанкционированного доступа. Логи доступны через `pm2 logs`.
- **Гибкая настройка безопасности**: Поддержка работы в локальных сетях без HTTPS (через `USE_SECURE_COOKIES`).

### 📋 Функциональность Kanban (v2.2)

### 📋 Функциональность Kanban
- **Подзадачи**: Полноценное создание и управление подзадачами внутри каждой карточки.
- **WIP-лимиты**: Ограничение количества задач в колонке для предотвращения перегрузки. Визуальное предупреждение при превышении.
- **Время цикла (Cycle Time)**: Автоматическое отслеживание времени нахождения задачи в работе и времени выполнения.
- **Swimlanes (Дорожки)**: Группировка задач по приоритетам с помощью умного поиска (введите `#priority`).
- **История активности**: Детальный лог перемещений и изменений каждой задачи.
- **Архивация**: Скрытие выполненных задач с возможностью восстановления или окончательного удаления.

### ✨ Интерфейс и UX
- **Deep Dark Premium Theme**: Новый элегантный темный режим с глубокими цветами и улучшенной читаемостью.
- **Улучшенный Glassmorphism**: Тонко настроенный эффект матового стекла, "светящиеся" границы и современные тени.
- **Background Picker**: Встроенные премиальные градиенты для персонализации рабочего пространства.
- **Markdown**: Полная поддержка Markdown в описании задач (жирный текст, списки, ссылки, код).
- **SortableJS**: Плавное и отзывчивое перетаскивание задач и колонок.
- **Celebration**: Эффект конфетти при завершении задач.
- **Smart Search**: Мгновенная фильтрация по названию, тегам и описанию с улучшенным контрастом.
- **Дедлайны**: Пульсирующая индикация для срочных задач (сегодня/завтра).

### 🔐 Безопасность и Надежность
- **Rate Limiting**: Защита от подбора пароля (ограничение попыток входа).
- **Environment Config**: Настройка через `.env` файл (секреты, порты).
- **Авто-бекапы**: Ежедневное резервное копирование данных с автоматической очисткой старых копий (хранение за последние 7 дней).
- **Безопасные сессии**: Куки с защитой `HttpOnly` и `SameSite`.

## Установка

### Требования
- Node.js (версия 16 или выше)

#### Как установить Node.js на Linux (ручной способ):
**Для Debian/Ubuntu/DietPi:**
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**Через NVM (рекомендуется):**
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
```

### Быстрая установка (через скрипт)
Вы можете установить приложение одной командой, которая автоматически скачает и запустит установщик:

```bash
curl -sSL https://raw.githubusercontent.com/iwizard7/KanBe/main/install.sh | bash
```

Установщик задаст вам несколько вопросов:
1.  **Путь установки** (по умолчанию текущая директория)
2.  **Пароль администратора** (для входа в веб-интерфейс)
3.  **Порт приложения** (скрипт проверит, свободен ли порт 3000, и предложит выбрать другой, если он занят)

Или вручную:
```bash
chmod +x install.sh
./install.sh
```
Скрипт сам установит зависимости, сгенерирует секретные ключи, настроит ваш пароль и создаст `.env` файл. Поддерживаются **Linux** и **macOS**.
## Развертывание в Nomad

Проект подготовлен для запуска в кластере **HashiCorp Nomad**.

### Быстрый старт с Docker
1. Соберите образ:
   ```bash
   docker build -t your-registry/kanbe:latest .
   ```
2. Отправьте его в ваш реестр:
   ```bash
   docker push your-registry/kanbe:latest
   ```

### Настройка в Nomad
В корне проекта находится файл `kanbe.nomad`. Он настроен на использование **Host Volumes** для постоянного хранения данных.

1. Убедитесь, что на ноде Nomad настроен том с именем `data` (например, в `/etc/nomad.d/client.hcl`):
   ```hcl
   client {
     host_volume "data" {
       path      = "/mnt/nomad-volumes"
       read_only = false
     }
   }
   ```
2. Запустите задачу:
   ```bash
   nomad job run kanbe.nomad
   ```

Данные будут сохраняться в `/mnt/nomad-volumes/kanbe` (или по вашему пути), что гарантирует их сохранность при перезагрузке контейнеров.

## API Endpoints

### Аутентификация
- `POST /api/login` — Вход (с защитой Rate Limit).
- `POST /api/logout` — Выход.
- `GET /api/check-auth` — Статус авторизации.

### Доска и Колонки
- `GET /api/board` — Данные доски.
- `POST /api/columns` — Новая колонка (с WIP-лимитом).
- `PUT /api/columns/:id` — Обновить заголовок или WIP.
- `DELETE /api/columns/:id` — Удалить колонку.

### Задачи
- `POST /api/columns/:columnId/tasks` — Создать задачу.
- `PUT /api/tasks/:taskId` — Обновить данные.
- `POST /api/tasks/:taskId/move` — Переместить (обновляет Cycle Time и историю).
- `POST /api/tasks/:taskId/archive` — Архивация/Восстановление.
- `GET /api/tasks/archived` — Список архивных задач.
- `GET /api/tasks/:taskId/history` — Лог действий задачи.

### Проекты
- `GET /api/projects` — Список всех проектов.
- `POST /api/projects` — Создать новый проект (имя, цвет).
- `PUT /api/projects/:id` — Обновить проект.
- `DELETE /api/projects/:id` — Удалить проект (снимает привязку у всех задач).


## Восстановление пароля

Если вы забыли пароль, но имеете доступ к серверу через SSH, есть несколько способов его сбросить:

### Способ 1: Использовать скрипт сброса (рекомендуется)

```bash
# Показать справку
node reset-password.js --help

# Интерактивный режим (с запросом пароля)
npm run reset-password
# или
node reset-password.js

# Неинтерактивный режим (прямое указание пароля)
node reset-password.js --password "your-new-password"
```

### Способ 2: Вручную через Node.js

```bash
node -e "const bcrypt = require('bcryptjs'); const fs = require('fs'); const hash = bcrypt.hashSync('admin', 10); fs.writeFileSync('data/config.json', JSON.stringify({passwordHash: hash}, null, 2)); console.log('✅ Пароль сброшен на: admin');"
```

### Способ 3: Редактировать config.json напрямую

1. Сгенерируйте новый хеш:
```bash
node -e "console.log(require('bcryptjs').hashSync('your-password', 10))"
```

2. Вставьте полученный хеш в `data/config.json`:
```json
{
  "passwordHash": "вставьте_хеш_здесь"
}
```

3. Перезапустите приложение

**⚠️ Важно**: После сброса пароля рекомендуется сменить его на более надежный через интерфейс приложения.

## Настройка окружения (Advanced)

Проект использует файл `.env` для конфигурации. Основные переменные:

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `PORT` | Порт, на котором запускается сервер | `3000` |
| `SESSION_SECRET` | Секретный ключ для шифрования сессий | Генерируется автом. |
| `NODE_ENV` | Режим работы (`production` / `development`) | `production` |
| `USE_SECURE_COOKIES` | Требовать HTTPS для куки (`true`/`false`) | `false` |
| `BACKUP_DAYS` | Количество дней хранения бекапов | `7` |

### Логирование
Приложение пишет подробные логи (включая IP адреса при аутентификации). 
Просмотр логов через PM2:
```bash
pm2 logs kanbe
```
Или напрямую в файлах `logs/combined.log` и `logs/error.log`.

## Резервное копирование
Бекапы создаются автоматически в `data/backups/`. Срок хранения настраивается в `.env` переменной `BACKUP_DAYS`.
